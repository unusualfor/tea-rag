# tea-rag

A personal tea collection explorer powered by RAG (Retrieval-Augmented Generation). Browse, filter, and search ~23 curated teas from Japan, China, India, and beyond.

## Stack

- **Runtime**: Cloudflare Workers
- **Vector store**: Cloudflare Vectorize (768-dim, BGE-base-en-v1.5)
- **Embeddings**: Cloudflare Workers AI (free tier)
- **Frontend**: React + Vite + shadcn/ui + Tailwind CSS
- **Backend**: Hono (TypeScript)
- **Storage**: R2 (photos), YAML files (tea data)

## Project structure

```
tea-rag/
├── content/teas/        # Source of truth: one YAML per tea
├── src/
│   ├── client/          # React frontend (Vite)
│   ├── worker/          # Hono API (Cloudflare Worker)
│   ├── shared/          # Shared TypeScript types
│   └── data/            # Compiled teas.json (generated by ingest)
├── scripts/
│   └── ingest.ts        # Parse YAML → validate → generate embeddings payload
├── wrangler.toml        # Cloudflare Workers config
├── DECISIONS.md         # Architecture decision log
└── package.json
```

## Getting started

```bash
# Install dependencies
npm install

# Parse all tea YAML files, validate, and generate data
npm run ingest

# Start dev server (frontend on :5173, API via wrangler on :8787)
npm run dev          # Vite dev server (frontend)
npx wrangler dev     # Worker dev server (API) — separate terminal
```

## How to add a new tea

1. Create a YAML file at `content/teas/{id}.yaml` using an existing file as template.
2. Run `npm run ingest` to validate and regenerate `src/data/teas.json`.
3. (Optional) Upload a photo to R2: `npx wrangler r2 object put tea-photos/photos/{id}.jpg --file ./path/to/photo.jpg`
4. Commit both the YAML file and the updated `teas.json`.
5. Deploy: `npm run deploy` (or push to `main` for auto-deploy).

To push embeddings to Vectorize, deploy the worker and call the ingest endpoint:

```bash
curl -X POST https://tea.francescoforesta.com/api/ingest \
  -H "Authorization: Bearer $INGEST_SECRET" \
  -H "Content-Type: application/json" \
  -d @dist/ingest-payload.json
```

## How to remove a tea

1. Delete the YAML file from `content/teas/`.
2. Run `npm run ingest` to regenerate data.
3. Delete the vector from Vectorize via the delete endpoint:
   ```bash
   curl -X POST https://tea.francescoforesta.com/api/ingest/delete \
     -H "Authorization: Bearer $INGEST_SECRET" \
     -H "Content-Type: application/json" \
     -d '{"ids": ["tea-id-to-remove"]}'
   ```
4. Commit and deploy.
